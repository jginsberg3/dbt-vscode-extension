# dbt Navigator

A VS Code extension for dbt development. Provides go-to-definition, autocompletion, hover documentation, DAG visualization, and multi-project support — all powered by your project's `manifest.json`.

## Features

### Autocompletion

- **Model names inside `ref()`** — type `ref('` and see all models from your manifest, with descriptions, materialization type, and column details
- **Source names inside `source()`** — two-phase completion: source names first, then table names filtered to the selected source
- **Jinja function snippets** — inside `{{ }}` blocks, get completions for `ref()`, `source()`, `config()`, `var()`, `env_var()`, `is_incremental()`, and more with tab-stop support
- **YAML schema properties** — context-aware completions in `.yml` files for model, column, source, and source table properties

### Hover Documentation

Hover over `ref('model_name')` or `source('source_name', 'table_name')` to see a rich popup with:
- Model/source description
- Materialization type and database location
- Column names, types, and descriptions (in a formatted table)

### Go-to-Definition

Cmd/Ctrl+Click on any `ref('model_name')` to jump directly to that model's SQL file.

### DAG Visualization

An interactive graph view in the bottom panel showing your project's dependency DAG. The graph highlights the currently open model and updates as you navigate between files.

### Multi-Project Support

Automatically discovers all dbt projects in your workspace (via `dbt_project.yml` files). Switch between projects with the `dbt: Switch Project` command or let the extension auto-switch when you open a file from a different project.

## Requirements

- VS Code 1.85.0 or later
- A dbt project with a compiled `manifest.json` (run `dbt compile` to generate one)
- For `.sql` syntax highlighting with Jinja support, you may need to install an extension that registers the `jinja-sql` language (ex: https://marketplace.visualstudio.com/items?itemName=samuelcolvin.jinjahtml)

## How It Works

The extension reads dbt's `manifest.json` file (generated by `dbt compile`) to understand your project's models, sources, columns, and dependency graph. It lazy-loads the manifest when a project becomes active and watches for changes — so running `dbt compile` automatically refreshes all completions and hover data.

No language server is used. All features are implemented directly with VS Code's `CompletionItemProvider`, `HoverProvider`, and `DefinitionProvider` APIs.

## Configuration

| Setting | Default | Description |
|---------|---------|-------------|
| `dbtNavigator.dbtCommand` | `"auto"` | The command used to run dbt. `"auto"` searches `.venv/bin`, `venv/bin`, then falls back to PATH. Set a custom value like `"/path/to/dbt"` to override. |

## Build the Extension
1. Run `npm install`
2. Run `npm run compile`
3. Run `npx vsce package`
4. Install the `.vsix` file (replace x with latest version): 
```
code --install-extension dbt-navigator-x.x.x.vsix
```

## Getting Started - Development

1. Clone this repo and open it in VS Code
2. Install extension dependencies:
   ```bash
   cd extension
   npm install
   npm run compile
   ```
3. Press **F5** to launch the Extension Development Host
4. Open a workspace containing a dbt project
5. Run `dbt compile` in your project (or use the `dbt: Compile Project` command) to generate `manifest.json`
6. Start editing `.sql` and `.yml` files — completions, hover, and go-to-definition will activate automatically

## Development

From the `extension/` directory:

```bash
npm install          # Install dependencies
npm run compile      # Build the extension
npm test             # Run tests (78 tests, ~120ms)
npm run test:watch   # Run tests in watch mode
npm run watch        # Recompile on file changes
```
